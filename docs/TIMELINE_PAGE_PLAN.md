# 时间轴 / 历史进度页规划

> 目标：新增一个“按时间回顾任务”的现代化页面，让 Recall 不只是记待办，也能回看阶段性成果和遗留事项。

## ✅ 当前进度（v0.3.3 原型起步）

- ✅ 已添加入口（侧边栏工具区新增「时间轴」）
- ✅ 已添加页面骨架（Timeline 原型视图，占位实现）

> 说明：当前只做了入口和页面骨架，后续按 Phase 1 逐步补齐分组、筛选、折叠、动效。

## 你想要的效果（整理后）

- 以时间轴形式展示各个时期的待办事项
- 能区分：
  - 已完成
  - 未完成
  - 已过期
  - 已过去 / 已归档
- 长内容可以折叠 / 展开
- 布局可以探索：
  - 时间轴 + 卡片流
  - 时间轴 + 瀑布流
- 颜色表达明确：
  - 已完成：绿色
  - 过期：红色
  - 已过去：灰色
  - 待办 / 进行中：主色系
- 整体风格要更现代、动态、有视觉层次

## 为什么这个页面值得做

这个页面能让 Recall 从“任务记录工具”变成“阶段进展可视化工具”。

价值在于：
- 看到自己某段时间做了什么
- 哪些任务拖延了
- 哪些任务完成得多
- 能形成“回顾感”和“成就感”

这很适合个人效率产品。

## 设计建议

### 布局方向
建议从 **“单列时间轴 + 卡片”** 开始，不要一上来就做瀑布流。

原因：
- 实现更稳
- 信息更清晰
- 和现有任务结构更容易兼容
- 后续再升级瀑布流也不晚

### 状态颜色建议
- 已完成：绿色系
- 已过期：红色系
- 已过去：灰色系
- 待办 / 进行中：主题主色系

### 动效建议
- 时间节点淡入
- 卡片 hover 提升
- 展开折叠平滑过渡
- 滚动时保留轻微“时间流动感”

原则：
**动态可以有，但不能影响阅读。**

## 技术实现建议

### 数据层
尽量复用现有 Task 数据，不额外造一套新模型。

需要补的只是：
- 时间分组逻辑
- 状态映射逻辑
- 折叠/展开 UI 状态

### 组件层
未来可拆成：
- `TimelinePage`
- `TimelineGroup`
- `TimelineCard`
- `TimelineFilters`
- `TimelineSummary`

### 版本建议
适合放在：
- `v0.3.3`：原型版上线
- `v0.3.4`：体验增强与动态效果完善

## 完成标准

这个页面至少要满足：
- 一眼能看懂时间顺序
- 一眼能区分完成 / 未完成 / 过期 / 已过去
- 长内容不会把页面撑爆
- 滚动体验顺畅
- 和现有主题系统风格一致

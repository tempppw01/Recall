# v0.3.2 Candidate Scope

> 目的：明确下一次 GitHub 正式发布 `v0.3.2` 的候选范围，避免内部迭代线与外部 release 线混淆。

## 当前目标

将最近几轮已经落地、且用户可感知的体验增强收敛为一次正式发布。

## 当前优先纳入项

### 1. Review / 检查页增强
- Review 工作流骨架
- Review 自动推进
- Review 进度指示
- Review 状态反馈层收口（目标中）

### 2. Quadrant / 四象限页收口
- 顶部 summary
- 象限层级统一
- 拖拽反馈增强
- 空态 / 折叠态收口

### 3. 与以上改动直接相关的稳定性修复
- Review 构建修复
- 类型签名修复
- 相关审计与流程收口文档

## 当前暂不纳入项

- 与当前用户感知较弱、仅面向内部治理的流程文档增强
- 尚未完成用户界面闭环的实验性改动
- 与 `Review / Quadrant` 主线无关的零散修补

## 当前发版门槛

`v0.3.2` 默认至少满足：
1. Review 页状态反馈层已收口
2. Quadrant 页已完成独立验收并已推送
3. 所有纳入项都有对应 audit
4. release note 能清楚描述“用户得到了什么增强”

## 当前判断

- Quadrant：**已基本满足纳入条件**
- Review：**还差状态反馈层最后一轮收口**
- 因此：**v0.3.2 还未最终 ready，但方向已经明确**

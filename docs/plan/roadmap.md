# Roadmap（0.1 → 1.0）

> 全局路线图：每个版本包含目标、范围、退出条件、风险与测试/可观测性要求。

## v0.1（MVP 稳定化）

目标：核心流程可用 + 关键边界收敛 + 可以可靠部署。

范围（示例）：
- AI 管理助手“直接操作”闭环
- 动态 PG / Prisma 边界收敛（默认关闭、受控启用）
- 时间轴 Phase 2（可读性、定位感、基础动效）
- 基础发布流程 + Docker 推送

退出条件：
- `docs/plan/versions/v0.1.md` 全部 ✅
- 最近一轮审计无“未处理高危问题”
- CHANGELOG 与 GitHub Release 一致

风险：
- Prisma migrations 未纳入版本控制会影响生产升级

测试/可观测性：
- `npm run typecheck` / `npm run lint` 必须通过
- Docker 能启动（最小健康检查：能响应首页或关键 API 之一）

## v0.2（体验增强 + 可维护性提升）

目标：在不引入复杂度爆炸的前提下，提升“扫描效率”和“成就感反馈”。

范围（示例）：
- 时间轴 Phase 3：
  - 瀑布流/更强视觉分层
  - 热力/完成密度
  - 阶段总结（本周/本月）
  - 主题色/渐变联动
- 迁移策略固化：引入 Prisma migrations，并提供生产 deploy 手册

退出条件：
- `docs/plan/versions/v0.2.md` 全部 ✅
- 性能回归不超过阈值（需要定义）

## v0.3（多端同步/可靠性）

目标：同步与冲突处理更可靠，减少“我明明做了但没同步上”的不确定性。

## v1.0（稳定发布 + UI 系统化）

目标：可作为长期使用的个人效率工具：稳定、可维护、易部署、文档完善，并拥有统一的 UI 规范系统。

范围（示例）：
- 建立统一 UI 规范系统（token / 组件 / 页面层级 / 扩展流程 / 动效规范）
- 关键页面按统一规范收敛（任务页 / 时间轴 / 设置页 / AI 面板），并提升动效质感
- 明确“新增功能优先复用 UI 规范”的开发约束
- 稳定发布要求（部署 / 回滚 / 监控 / 文档）完善

退出条件：
- `docs/plan/versions/v1.0.md` 全部 ✅
- UI / Motion 规范文档可指导新功能扩展，而不是停留在概念层
- 至少 3 个关键页面完成规范化回收

风险：
- 如果只补文档不回收现有页面，规范会迅速失效
- 如果 token / 组件 / motion 边界不清，会继续出现“样式散点化”和“动画廉价感”

测试/可观测性：
- 关键页面在主题切换 / 响应式下表现一致
- 新功能 PR 需说明复用了哪些 token / 组件规范


## v1.1（交互质感与移动端可达性）

目标：在 v1.0 规范基础上，继续提高“动画高级感”和移动端真实可用性。

范围（示例）：
- 时间轴 / 任务卡片 / 设置弹窗的第二波动效打磨
- 固定底栏 + 滚动容器的安全区统一策略
- 关键移动端页面可达性回归（最后一项可见、可点、可滚）

退出条件：
- `docs/plan/versions/v1.1.md` 全部 ✅
- 最近审计无未处理高危问题
- 至少 2 个“用户可明显感知”的交互细节升级


## v1.2（默认行为一致性与体验回归）

目标：把近期高频反馈收敛为可持续的默认行为与回归基线。

范围（示例）：
- 默认首页、侧栏展开状态等导航偏好持久化
- 天气定位体验（成功/失败路径）优化
- AI 管理助手在不同配置场景的调用稳定性回归
- 移动端布局与触达回归清单

退出条件：
- `docs/plan/versions/v1.2.md` 全部 ✅
- 最近审计无未处理高危问题


## v1.3（同步可靠性与回归压测）

目标：减少同步链路不确定性，并建立可复用的回归压测基线。

范围（示例）：
- 云同步入队/轮询链路的敏感参数与错误提示收敛
- AI 助手核心路径（manage-agent/todo-agent）回归压测
- 移动端关键页面实机抽测记录化

退出条件：
- `docs/plan/versions/v1.3.md` 全部 ✅
- 最近审计无未处理高危问题

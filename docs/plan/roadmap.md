# Roadmap（v0.1 → v0.2 → v0.3 → v1.0）

> 全局路线图：每个版本都要包含目标、范围、退出条件、风险与测试/可观测性要求。
> 当前执行版本固定为 `v0.1`；在 `docs/plan/versions/v0.1.md` 全部 ✅ 且满足退出条件前，后续版本仅做规划。

## v0.1（MVP 稳定化）

目标：核心流程可用、关键边界收敛、可以可靠部署。

范围：
- AI 管理助手“直接操作”闭环
- 动态 PG / Prisma 边界收敛（默认关闭、受控启用）
- 时间轴 Phase 2（可读性、定位感、基础动效）
- 基础发布流程 + Docker 推送
- 计划目录统一到 `docs/plan/`

退出条件：
- `docs/plan/versions/v0.1.md` 全部 ✅
- 最近一轮审计无“未处理高危问题”
- `docs/plan/releases/CHANGELOG.md` 与 Release notes 一致

风险：
- 仓库历史上已有 `v0.2.x` / `v0.3.x` tag，和当前强制流程存在历史不一致，需要在后续发版时明确说明
- Prisma migrations 若未纳入版本控制，会影响生产升级稳定性

测试/可观测性：
- `npm run typecheck` / `npm run lint` 通过
- Docker 最小健康检查可用（首页或关键 API 可访问）
- 发布前核对 changelog、tag、release notes 一致

改进建议：
- 高：补齐 v0.1 发布校验清单与历史版本说明模板
- 中：把历史散落计划继续收口为 archive 或指针文档
- 低：为审计报告增加“已修复 / 待处理”字段模板

## v0.2（体验增强 + 可维护性提升）

目标：在不引入复杂度爆炸的前提下，提升扫描效率和成就感反馈。

范围：
- 时间轴 Phase 3：
  - 更强视觉分层
  - 热力 / 完成密度
  - 阶段总结（本周 / 本月）
  - 主题色 / 渐变联动
- 迁移策略固化：引入 Prisma migrations，并提供生产 deploy 手册
- 持续整理计划与发布文档结构

退出条件：
- `docs/plan/versions/v0.2.md` 全部 ✅
- 最近一轮审计无未处理高危问题
- 性能回归不超过既定阈值

风险：
- 如果体验增强没有统一规范，容易继续堆出样式碎片
- 发布编号若继续沿用历史 tag，需额外说明版本语义

测试/可观测性：
- 体验改动需补充最小回归清单
- 新增部署/迁移步骤需可脚本化验证

改进建议：
- 高：给时间轴增强建立明确性能预算
- 中：补齐生产迁移与回滚手册
- 低：增加体验埋点便于后续判断功能真实使用率

## v0.3（多端同步 / 可靠性）

目标：同步与冲突处理更可靠，减少“做了但没同步上”的不确定性。

范围：
- sync 模型与冲突策略明确化
- 错误码、requestId、结构化日志收敛
- 深度健康检查与同步诊断增强

退出条件：
- `docs/plan/versions/v0.3.md` 全部 ✅
- 最近一轮审计无未处理高危问题
- 关键同步链路具备最小回归脚本

风险：
- 同步链路跨客户端、跨环境，真实问题更偏边界场景
- 日志字段若不统一，定位成本会迅速上升

测试/可观测性：
- 至少覆盖成功 / 冲突 / 超时 / 凭证异常四类同步场景
- 健康检查输出需可被人工快速判读

改进建议：
- 高：补充冲突复现样例与处理手册
- 中：增加同步状态提示的一致性设计
- 低：为诊断输出增加分级文案

## v1.0（稳定发布 + UI 系统化）

目标：可作为长期使用的个人效率工具：稳定、可维护、易部署、文档完善，并拥有统一 UI 规范系统。

范围：
- 建立统一 UI 规范系统（token / 组件 / 页面层级 / 扩展流程 / 动效规范）
- 关键页面按统一规范收敛（任务页 / 时间轴 / 设置页 / AI 面板）
- 新增功能优先复用 UI 规范
- 稳定发布要求（部署 / 回滚 / 监控 / 文档）完善

退出条件：
- `docs/plan/versions/v1.0.md` 全部 ✅
- UI / Motion 规范可直接指导新功能扩展
- 至少 3 个关键页面完成规范化回收

风险：
- 只补文档、不回收现有页面，会让规范很快失效
- token / 组件 / motion 边界不清会继续导致样式散点化

测试/可观测性：
- 关键页面在主题切换 / 响应式下表现一致
- 新功能 PR 必须说明复用了哪些 token / 组件规范

改进建议：
- 高：建立 UI 回归基线截图或检查清单
- 中：将常见组件抽成更稳定原语
- 低：为动效建立设计语言示例库

## 1) 漏洞/安全
- 高危：无
- 中危：
  - GET 轮询阶段过去仅允许服务端 env，不接受客户端 Redis 参数，导致“POST 能入队、GET 无法轮询”断链。
  - 处理：GET 增加 query 参数解析，并复用 resolveRedisConfig 优先级（env > fallback）。
- 低危：
  - Redis 密码经 query 透传时可能出现在中间层日志（部署侧风险）。建议后续切为 job-token 绑定配置，减少 query 明文。

# Audit — 2026-03-21 Round 70

## 本轮目标
将“清除已完成”从浏览器原生确认框改为应用内统一确认弹窗，避免 UI 风格割裂。

## 本轮不做
- 不改其他 `window.confirm`
- 不重构全局 modal / dialog 基础设施
- 不改 `clearCompletedTasks()` 的业务逻辑
- 不扩展成通用批量删除确认体系

## 实际改动
- 只修改 `src/app/page.tsx`
- 为“清除已完成”新增页内状态控制的确认弹窗
- 将 `PageTopBar` 的 `onClearCompleted` 从 `window.confirm(...)` 改为打开应用内弹窗
- 保持原有清除逻辑不变：确认后仍调用 `clearCompletedTasks()`，取消则无副作用
- 弹窗视觉沿用现有玻璃拟态 modal 体系，主次按钮清晰区分

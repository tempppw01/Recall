# 2026-03-21 Round 69 — Timeline task card modernization

## 本轮目标
- 只优化时间轴页中日分组下的任务卡片展现形式
- 提升现代感、层级感和扫读效率
- 不改数据结构、不改分组逻辑、不扩成功能重构

## 产品收益句
- 本轮完成后，用户将更容易在时间轴里快速扫读任务状态，并愿意继续往下浏览。

## 多 Agent 参与
- pm-coder：定义本轮最小范围、现代感问题与验收标准
- ui-designer：给出卡片层级、轻量动效与视觉克制建议
- code-helper：给出最小文件集合、低风险改法与交互风险点
- qa-tester：末段做技术验收 + 产品验收
- main：主控落代码、build、audit、commit、push

## 实际修改
- 文件：`src/app/components/timeline/TimelinePanel.tsx`
- 任务卡片从旧的“状态点 + emoji badge + 厚重块面”改成：
  - 左侧状态光带
  - 更干净的状态 badge（去掉 emoji）
  - 三层信息布局：主信息 / 时间信息 / 辅助信息
  - 更轻的卡片表面、hover 浮起、阴影过渡
  - 标签改为更轻的辅助 chip
  - 长标题展开按钮文案改为更明确的“展开详情 / 收起详情”

## 验证
- `npm run build` ✅

## 产品判断
- 本轮没有扩题到 summary / heatmap / 筛选区
- 用户在同一天多张任务卡之间的连续浏览体验更轻、更规整
- 任务状态表达不再依赖 emoji，整体更像正式产品界面

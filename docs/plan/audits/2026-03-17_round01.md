## 1) 漏洞/安全

### 高危
- 无（本轮未发现可直接远程触发的高危漏洞）。

### 中危
- Dockerfile 中存在 `ENV NEXTAUTH_SECRET="change-me-in-prod"`。
  - 风险：误用默认值会导致生产环境会话安全性下降。
  - 现状：已有注释提醒生产覆盖；但仍建议在运行时强制检查（启动时若仍为默认值则 warn 或拒绝启动）。
  - 建议：加入启动时校验（例如在 server 启动时检测 NEXTAUTH_SECRET 是否为默认值）。

- 动态 PG（x-pg-*）模式属于扩大攻击面功能。
  - 现状：已收敛为默认关闭 + 仅未登录生效 + 支持 token/allowlist。
  - 建议：文档与设置页进一步强化提示，默认引导配置 `PG_HEADERS_TOKEN`。

### 低危
- Docker Compose 示例默认提供弱口令（postgres/postgres）。
  - 建议：README/compose 中强调生产必须改。

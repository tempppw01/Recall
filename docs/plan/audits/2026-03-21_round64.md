# Round 64 Audit — Quadrant 四象限页 UI 收口

- 日期：2026-03-21
- 范围：仅收口 `src/app/page.tsx` 中 `activeFilter === 'quadrant'` 分支，不扩到其他页面，不修改四象限算法与任务业务逻辑。

## 本轮目标

按协作要求完成一个小步：
1. 为 Quadrant 增加独立顶部区（标题、说明、总览摘要）。
2. 统一四个象限容器层级与面板样式，优先复用 `glass-panel` / `glass-panel-soft` / 现有 token。
3. 增强拖拽悬停反馈，让目标象限在 `onDragOver` 时有明确高亮。
4. 改善空态与折叠态文案。
5. 不修改四象限算法或任务业务逻辑。

## 实际改动

### 1) 增加 Quadrant 顶部区
- 新增独立 header panel。
- 展示：
  - `Quadrant / 四象限` 标题
  - 一句使用说明
  - 总览摘要：未完成任务数、重要任务数、紧急任务数
  - 当前任务堆积最多的象限提示

### 2) 统一四象限面板样式
- 外层四象限卡片统一改为 `glass-panel`。
- 卡片内任务区域统一为 `glass-panel-soft`。
- 边框、圆角、阴影、层级均复用现有 token / 语义 surface，避免新增并行样式体系。

### 3) 拖拽悬停高亮
- 新增 `dragOverQuadrantKey` 状态，仅用于 UI 反馈。
- 在象限容器的 `onDragEnter` / `onDragOver` / `onDragLeave` / `onDrop` 中维护高亮态。
- 高亮时增强：
  - 边框强调
  - 背景渐变强调
  - 阴影强调
  - 说明文案切换为“松手即可放到这里”
- `onDrop` 后清理高亮态。

### 4) 空态与折叠态文案优化
- 空态从“暂无任务”升级为更明确的引导文案。
- 拖拽命中空态时，额外提示“按当前象限规则自动更新优先级 / 截止时间”。
- 折叠态改为：
  - “已先展示前 X 项，剩余 Y 项可展开查看”
  - 展开后提示“当前已展开完整列表，处理完后可以收起”
- 折叠按钮文案改为“展开剩余 X 项 / 收起长列表”。

## 明确未改内容

- 未修改四象限判定逻辑：
  - `isImportantTask`
  - `isUrgentTask`
  - `quadrantGroups` 的分类算法
- 未修改拖入象限后的业务规则：
  - `moveTaskToQuadrant`
  - 任务优先级 / dueDate 的处理逻辑保持原样
- 未修改其他页面或组件文件

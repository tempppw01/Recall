## 2) Bug / 边界条件

- 高危：无。
- 中危：
  - Review 页现在自行维护 `focusedTaskId`，与外层 `selectedTask` 共同作用；这是为了解决“处理完当前项后自动推进下一项”的体验问题，但后续如果主页详情区和 Review 页联动再增强，需要注意两套焦点来源的收敛规则。
  - 处理：本轮已用 effect 保持外部选择与内部焦点同步，并在当前分组清空时自动回退。
- 低危：
  - 处理动作后的成功提示目前使用轻量文本反馈，足够支撑当前节奏，但后续可以考虑统一成全局 toast / action feedback 机制。

- 高危：无。
- 中危：
  - Review 页这轮最大的收益不是样式，而是“连续处理感”建立：完成、延期、稍后再看之后，会自动切到下一项，工作流明显更顺。

- 低危：

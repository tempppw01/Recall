# 0.0.2 页面级执行清单

> 目标：把 0.0.2 从“方向性规划”拆成真正可执行的页面收敛任务。
> 本阶段只处理“结构与视觉收敛”，不引入新的复杂工作流页面。

## 一、阶段目标

0.0.2 要解决的不是“功能不够多”，而是：

- 页面结构不统一
- 层级不够清楚
- 新建任务样式太生硬
- 深色主题不够稳定
- 导航、Header、内容区之间缺少统一语言

一句话：
**先把 Recall 从“能用”整理到“顺眼、顺手、结构清楚”。**

## 二、页面优先级

### P0（最优先）
1. 左侧导航 / 全局布局框架
2. 任务主页面
3. 新建任务卡片

### P1
4. 时间轴页面
5. 页面通用 Header
6. 通用卡片 / 列表项

### P2
7. 空态 / 加载态 / 错态
8. 设置页视觉收敛

## 三、页面级任务拆分

## 1. 左侧导航 / 全局布局框架

### 目标
让导航真正承担“产品信息架构入口”的作用，而不是一列普通按钮。

### 任务
- [x] 重新定义导航分区（AI / 快捷入口 / 功能 / 列表等已完成第一轮收敛）
- [x] 统一导航项的图标、文字、计数、选中态
- [x] 强化当前所在页面的识别度
- [x] 统一侧栏背景、边框、分组间距
- [x] 检查折叠/展开后的可理解性（第一轮完成）

### 验收标准
- 用户进入后能快速理解产品结构
- 当前页面一眼可辨认
- 导航在深色主题下层级稳定、不过乱

## 2. 任务主页面

### 目标
让任务页从“信息堆在一起”变成“结构清楚、重点明确”的主工作台。

### 任务
- [x] 统一任务页顶部区域（标题 / 说明 / 快捷操作）
- [x] 统一筛选区、工具区、列表区的顺序
- [x] 优化任务项的标题、元信息、状态信息排布
- [x] 拉开父任务 / 子任务 / 状态 / 标签之间层级
- [x] 减少视觉噪音，让重点信息前置

### 验收标准
- 用户能快速扫读任务列表
- 高信息密度下仍能分清重点
- 结构比当前版本更稳定、更像工作台

## 3. 新建任务卡片

### 目标
去掉当前“直接新建”的生硬样式，改成完整、自然、和任务卡片一致的创建入口。

### 任务
- [x] 将新建任务入口改为卡片式创建
- [x] 容纳标题、备注、日期、优先级、标签等常用入口（第一轮）
- [x] 保证创建态和已有任务卡片属于同一视觉系统
- [x] 优化聚焦、输入、保存、取消的状态反馈（第一轮）
- [x] 避免新建态像一条突兀插入的编辑条

### 验收标准
- 新建体验更像“创建一张任务卡片”
- 创建态不突兀，不破坏列表节奏
- 专业感明显提升

## 4. 时间轴页面

### 目标
保留现有能力，但把结构、层级和视觉节奏继续收顺。

### 任务
- [x] 统一时间轴页 Header 与主页面风格
- [x] 优化日期分组、内容块、状态标签的排布
- [x] 统一卡片密度与留白节奏
- [x] 让“今天 / 当前位置 / 当前焦点”更明确
- [x] 检查深色主题下的可读性与层级感

### 验收标准
- 时间轴页面不显散、不显挤
- 阅读路径顺畅
- 和任务主页属于同一产品语言

## 5. 页面通用 Header

### 目标
统一所有页面顶部的标题语言和操作布局。

### 任务
- [x] 统一页面标题字号、字重、间距
- [x] 统一副标题 / 说明文字样式
- [x] 区分主要操作与次要操作
- [x] 统一 Header 和内容区之间的过渡关系

### 验收标准
- 不同页面切换时，顶部结构认知一致
- 页面标题区不再东一块西一块

## 6. 通用卡片 / 列表项

### 目标
建立基础视觉原语，避免每个页面各画各的。

### 任务
- [x] 统一主卡片 / 次卡片样式（主页 / 时间轴 / 设置 / modal 第一轮完成）
- [x] 统一列表项的间距、边框、hover、selected（任务 / 侧栏第一轮完成）
- [x] 统一状态标签（badge）风格（核心区域完成）
- [x] 统一操作按钮在卡片内的位置和视觉重量（核心区域完成）

### 验收标准
- 卡片和列表项有明显“同一家人”的感觉
- 新页面可直接复用，而不是重写一套样式

## 7. 空态 / 加载态 / 错态

### 目标
补齐细节体验，避免主页面收好了，边角还很粗糙。

### 任务
- [x] 统一空态文案与结构（主页 / 时间轴 / 日志等已完成第一轮）
- [ ] 统一加载骨架或轻量 loading 表达
- [ ] 统一错误提示与重试入口

### 验收标准
- 状态反馈语言一致
- 页面边缘场景不再显得像临时拼接

## 8. 设置页视觉收敛

### 目标
让设置页不再像独立风格页面，而是并入整个产品语言。

### 任务
- [x] 统一设置区块、标题、说明、表单间距
- [x] 统一按钮、输入框、说明文字风格
- [x] 统一分组逻辑与视觉层级

### 验收标准
- 设置页和主工作区看起来属于同一产品

## 四、推荐执行顺序

### 第一轮
- 左侧导航 / 全局布局框架
- 页面通用 Header
- 通用卡片 / 列表项

### 第二轮
- 任务主页面
- 新建任务卡片

### 第三轮
- 时间轴页面
- 空态 / 加载态 / 错态
- 设置页视觉收敛

## 五、交付物建议

0.0.2 真正开工时，建议每轮至少产出：

1. 页面改动
2. 审计报告
3. README 进度更新
4. 如有结构变化，同步更新 `ui-spec.md` 或 `ui-refactor-checklist.md`

## 六、一句话总结

0.0.2 不是拼命加功能。
0.0.2 的核心是：
**先把骨架、导航、卡片、创建体验、深色主题这几件事收顺。**

# v1.0 任务清单（仅规划，禁止开工）

> 前提：只有在 v0.x 路线按规则完成并发版后，才允许进入 v1.0。

## 目标

- 建立长期可维护的产品基线：稳定、统一、易扩展、易部署。

## 任务列表

### A) UI 系统化

- ⬜ 建立统一 UI 规范系统（DoD：token / 组件 / 页面层级 / 动效规范文档完整）
- ⬜ 统一关键页面 UI 原语（DoD：任务页 / 时间轴 / 设置页 / AI 面板复用同一套设计原语）
- ⬜ 建立“新增功能先走 UI 规范”的开发约束（DoD：规范已写入项目约束文档）

### B) 交互与发布稳定性

- ⬜ 统一关键交互动效系统（DoD：关键页面动效节奏一致，不显得廉价或突兀）
- ⬜ 补齐稳定发布要求（DoD：部署 / 回滚 / 监控 / 文档形成闭环）

## 候选改进建议

### 高优先级
- 建立 UI 回归基线与截图检查机制
- 明确原语复用与新增原语的评审门槛

### 中优先级
- 为动效和层级建立示例库
- 收敛跨页面状态反馈组件

### 低优先级
- 补充设计原则说明，方便新成员接入

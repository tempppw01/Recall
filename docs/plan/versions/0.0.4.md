# 0.0.4 任务清单（仅规划，禁止开工）

> 前提：只有在 0.0.3 完成并完成对应发版后，才允许切换到 0.0.4。

## 目标

- 完成整体 UI / 界面重构，把产品收敛成统一、成熟、专业的效率工具体验。

## 任务列表

### A) UI / 界面整体重构

- ⬜ 建立统一 UI 规范系统（DoD：token / 组件 / 页面层级 / 动效规范文档完整）
- ⬜ 统一关键页面 UI 原语（DoD：任务页 / 时间轴 / 设置页 / AI 面板复用同一套设计原语）
- ⬜ 启动整体 UI / 界面重构（DoD：形成页面清单、分阶段重构顺序、信息层级与视觉语言基线；参考 `docs/plan/ui-refactor-checklist.md`）
- ⬜ 建立“新增功能先走 UI 规范”的开发约束（DoD：规范已写入项目约束文档）

### B) 体验与发布稳定性

- ⬜ 统一关键交互动效系统（DoD：关键页面动效节奏一致，不显得廉价或突兀）
- ⬜ 补齐稳定发布要求（DoD：部署 / 回滚 / 监控 / 文档形成闭环）

## 候选改进建议

### 高优先级
- 建立 UI 回归基线与截图检查机制
- 先输出整站 UI 重构清单（哪些页面先改、哪些原语先统一）

### 中优先级
- 将常见组件抽成更稳定原语
- 为动效和层级建立示例库

### 低优先级
- 补充设计原则说明，方便新成员接入
